Use Case Scenario: Create Account


Context: The user has the system open and has not logged in or created an account already


Primary Flow:
1. The user clicks the “Create Account” button
2. The user enters a username
3. The user enters a password
4. The user enters a first name
5. The user enters a last name
6. The user clicks the “Confirm” button
7. The user is directed to the landing page


Alternative Flows:


The user enters a username that is already taken by another user
A7. The user is shown an error message stating that the entered username has already been taken
A8. The user enters valid credentials
A9. Start from Primary step 6


The user cancels the create account action
A2. The user clicks the “Cancel” button
A3. The user is directed to the login screen


The server failed to create the account
A7. The user is shown an error message concerning the server failing to add their credentials as a new user
A8. The user is directed to the login screen
