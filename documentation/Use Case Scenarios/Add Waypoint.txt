Use Case Scenario: Add Waypoint


Context: The user is logged in to the system


Primary Flow:
1. The user clicks on a trip
2. The user selects a date on the calendar in the range of the trip
3. The user clicks the “Create Waypoint” button
4. The user enters the time and place for the waypoint
5. The user clicks the "Create Waypoint" button to submit the new waypoint
6. The user is directed to the list of waypoints for the chosen date


Alternative Flows:


The user cancels the operation
A4. The user clicks the “Cancel” button
A5. The user is redirected to the list of waypoints for the chosen date
