Use Case Scenario: Login


Context: The user has the system open and is not already logged in


Primary Flow:
1. The user enters their credentials
2. The user clicks on the “Login” button
3. The user is taken to the landing page


Alternative Flows:


The user enters the wrong username
A1. The user enters the wrong username
A2. The user is shown an error message stating the username is incorrect and cannot log in


The user enters the wrong password
A1. The user enters the wrong password
A2. The user is shown an error message stating the username is incorrect and cannot log in


The user fails to enter a username or password
A1. The user clicks on the “Login” button
A2. The user is shown an error message that the fields must be filled out to log in
